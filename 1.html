Introduction to CPU Scheduling

CPU scheduling is the process of deciding which process in the ready queue should be assigned to the CPU for execution. Since multiple processes may be waiting, the operating system uses scheduling to manage CPU time effectively. It helps in improving system efficiency, reducing process waiting time, and ensuring fair resource sharing. Various algorithms like FCFS, SJF, Round Robin, and Priority Scheduling are used based on system needs and goals.

Functionalities of CPU Scheduling

  • Maximizes CPU Utilization
  • Reduces Waiting Time
  • Ensures Fair Process Execution
  • Minimizes Response Time
  • Balances Load Among Processes

CPU Scheduling Algorithms

1. First Come First Serve (FCFS)

First-Come, First-Served (FCFS) is the simplest CPU scheduling algorithm where the process that arrives first is executed first. It follows a non-preemptive approach, meaning once a process starts, it runs until completion.

2. Shortest Job First (SJF)

Shortest Job First (SJF) is a CPU scheduling algorithm that selects the process with the smallest execution time next. It can be preemptive or non-preemptive and helps reduce average waiting time effectively.

3. Longest Job First (LJF)

Longest Job First (LJF) is a CPU scheduling algorithm that selects the process with the longest execution time to run next. It is less common and can lead to longer waiting times for shorter processes.

4. Shortest Remaining Time First (SRTF)

Shortest Remaining Time First (SRTF) is the preemptive version of Shortest Job First scheduling. It selects the process with the least remaining execution time and can preempt a running process if a shorter one arrives.

5. Longest Remaining Time First (LRTF)

Longest Remaining Time First (LRTF) is the preemptive version of Longest Job First scheduling. It selects the process with the longest remaining execution time and can preempt the current process if a new process with more remaining time arrives.

6. Round Robin (RR)

Round Robin (RR) is a preemptive CPU scheduling algorithm where each process gets a fixed time slice (called a time quantum) to execute. If a process doesn't finish within that time, it is moved to the end of the queue, ensuring fairness among all processes.

7. Priority Scheduling (Non-Preemptive)

Priority Scheduling (Non-Preemptive) is a CPU scheduling algorithm where each process is assigned a priority, and the process with the highest priority is executed first. If two processes have the same priority, they are scheduled based on their arrival order. Once a process starts executing, it runs to completion without interruption.

8. Priority Scheduling (Preemptive)

Priority Scheduling (Preemptive) is a CPU scheduling algorithm where the process with the highest priority is always selected for execution. If a new process with a higher priority arrives, it preempts the currently running process, which is paused and placed back in the ready queue. This ensures that high-priority tasks are executed first.
