Priority (Non-Preemptive) Scheduling

Introduction to Priority (Non-Preemptive) Scheduling

Priority scheduling assigns a priority value to each process and the CPU is allocated based on these priorities. In non-preemptive priority scheduling, once a process starts executing, it continues until completion.

This algorithm is useful in environments where certain processes need preferential treatment, such as real-time systems or critical business applications.

Characteristics of Priority (Non-Preemptive)

  • Non-Preemptive: Once a process starts execution, it continues until completion.
  • Priority-Based: Higher priority processes are executed first (lower numbers typically indicate higher priority).
  • Starvation: Lower priority processes may never execute if higher priority processes keep arriving.
  • Aging: Can be implemented to gradually increase the priority of waiting processes.

How Priority (Non-Preemptive) Works

Priority scheduling selects the process with the highest priority from the ready queue. If multiple processes have the same priority, FCFS is typically used as a tie-breaker.

Example:

Consider three processes with different priorities (lower number = higher priority):

  • P1: Arrival: 0ms, Burst Time: 5ms, Priority: 3
  • P2: Arrival: 1ms, Burst Time: 3ms, Priority: 1
  • P3: Arrival: 2ms, Burst Time: 8ms, Priority: 2

Execution order: P1 (0-5ms) → P2 (5-8ms) → P3 (8-16ms)

Note: P1 starts first as it's the only process available at time 0. After P1 completes, P2 executes as it has higher priority than P3.
