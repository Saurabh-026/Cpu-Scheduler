Priority (Preemptive) Scheduling

Introduction to Priority (Preemptive) Scheduling

Priority (Preemptive) scheduling is similar to non-preemptive priority scheduling, but it allows for interruption when a higher priority process arrives.

This algorithm is particularly useful in real-time systems where immediate response to high-priority events is critical.

Characteristics of Priority (Preemptive)

  • Preemptive: The current process can be interrupted if a higher priority process arrives.
  • Priority-Based: Higher priority processes are executed first (lower numbers typically indicate higher priority).
  • Starvation: Lower priority processes may never execute if higher priority processes keep arriving.
  • Aging: Can be implemented to gradually increase the priority of waiting processes.
  • Context Switching Overhead: Higher overhead due to potential frequent preemption.

How Priority (Preemptive) Works

Priority (Preemptive) scheduling continuously monitors process arrivals. Whenever a new process arrives, its priority is compared with the currently executing process. If the new process has higher priority, the current process is preempted.

Example:

Consider three processes with different priorities (lower number = higher priority):

  • P1: Arrival: 0ms, Burst Time: 5ms, Priority: 3
  • P2: Arrival: 1ms, Burst Time: 3ms, Priority: 1
  • P3: Arrival: 2ms, Burst Time: 8ms, Priority: 2

Execution order: P1 (0-1ms) → P2 (1-4ms) → P3 (4-12ms) → P1 (12-16ms)

Note: P1 starts execution at time 0, but is preempted when P2 arrives at time 1 because P2 has higher priority. After P2 completes, P3 executes next as it has higher priority than P1.
