Shortest Job First (SJF) Scheduling

Introduction to SJF Scheduling

Shortest Job First (SJF) scheduling prioritizes the process with the smallest burst time. This algorithm aims to reduce the average waiting time for processes.

SJF is theoretically optimal for minimizing average waiting time but requires predicting burst times before execution.

Characteristics of SJF

  • Non-Preemptive: Once a process starts, it runs until completion.
  • Optimal: Provides minimum average waiting time among all scheduling algorithms.
  • Starvation: Long processes may never execute if shorter ones keep arriving.
  • Prediction Required: Needs to predict CPU burst time before execution.

How SJF Works

SJF selects the process with the smallest burst time from the ready queue. If multiple processes arrive simultaneously, the one with the shortest execution time is chosen first.

Example:

Consider three processes arriving at different times:

  • P1: Arrival: 0ms, Burst Time: 5ms
  • P2: Arrival: 1ms, Burst Time: 3ms
  • P3: Arrival: 2ms, Burst Time: 8ms

Execution order: P1 → P2 → P3 (P1 starts first as it arrives at time 0, then P2 as it has shorter burst time than P3)
