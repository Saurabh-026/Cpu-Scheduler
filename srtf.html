Shortest Remaining Time First (SRTF) Scheduling

Introduction to SRTF Scheduling

Shortest Remaining Time First (SRTF) is the preemptive version of SJF. It continuously selects the process with the shortest remaining burst time.

SRTF provides optimal average waiting time but with the overhead of frequent context switching and the need for continuous monitoring.

Characteristics of SRTF

  • Preemptive: The current process can be interrupted if a new process with shorter remaining time arrives.
  • Optimal: Provides minimum average waiting time among all scheduling algorithms.
  • Starvation: Long processes may suffer indefinite delays if shorter processes keep arriving.
  • Context Switching Overhead: Higher overhead due to frequent preemption.

How SRTF Works

SRTF continuously monitors the remaining time of all processes in the ready queue. Whenever a new process arrives, it compares its burst time with the remaining time of the current executing process. If the new process has a shorter burst time, the CPU switches to it.

Example:

Consider three processes arriving at different times:

  • P1: Arrival: 0ms, Burst Time: 5ms
  • P2: Arrival: 1ms, Burst Time: 3ms
  • P3: Arrival: 2ms, Burst Time: 8ms

Execution order: P1 (0-1ms) → P2 (1-4ms) → P1 (4-8ms) → P3 (8-16ms)
